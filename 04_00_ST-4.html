控制语句

控制语句是用来改变程序流程的，根据条件使处理流程分支，循环执行，改变执行的筛选和次数。

控制语句一览

控制语句分为以下所示的“条件分支语句”和“回圈控制语句”、“程式执行结束语句”,共三种。

▌条件分支语句

条件分支语句按照指定条件对处理流程进行分支。

控制语句

控制内容

IF 语句

（1） 根据条件式执行处理的分支控制语句

IF条件式ATHEN

　　处理语句 B;

END_IF;

条件式 A 成立时，

执行处理语句 B。

条件式 A  不成立时，不作任何处理。

（2）根据条件式执行单独处理的2分支控制

IF条件式ATHEN

　　处理语句 B;

ELSE

　　处理语句 C;

END_IF;

条件式 A 成立时，

执行处理语句 B。

条件式 A 不成立时，

执行处理语句 C。

（3）根据条件值分别进行处理的多分支控制语句

IF条件式ATHEN

　　处理语句 B;

ELSIF条件式CTHEN

　　处理语句 D;

　　・

　　・

　　・

END_IF;

条件式 A 成立时，

执行处理语句 B。

条件式 A 不成立而条件式 C 成立时，

执行处理语句D。

 

 

 

条件式A、C……都不成立时，不作任何处理。

CASE 语句

根据条件值选择对应的处理并执行的多分支控制语句

CASE比较源OF

条件值A:

　　处理语句 B；

条件值C, 条件值D:

　　处理语句 E;

条件值F..条件值G:

　　处理语句 H;

　　・

　　・

ELSE

　　处理语句 I;

END_CASE;

 

当比较源和条件值 A 相等时，

执行处理语句 B。

比较源等于条件值C或条件值D时，

执行处理语句E。

比较源大于等于条件值F、小于条件值G时，

执行处理语句H。

 

 

比较源不满足任何条件时，

执行处理语句I。

* :表示插入半角空格。

▌循环控制语句

循环控制语句在指定条件下重复执行。

控制语句

控制内容

FOR 语句

循环控制语句循环处理，直到指定次数的条件式成立为止

FOR变量 := 初始值TO最终值BY增量值DO

　　处理语句 ;

END_FOR;

在逐步递增变量值的同时循环执行处理语句。

当变量值超过（或低于）最终值时，结束循环。

WHILE 语句

在条件式成立（真）时，循环处理的不定次数循环控制语句

WHILE条件式DO

　　处理语句 ;

END_WHILE;

条件式成立时

循环执行处理语句

REPEAT语句

在条件式成立（变为真）之前，循环处理的不定次数循环控制语句

REPEAT

　　处理语句 ;

　　UNTIL条件式

END_REPEAT;

条件式不成立时

循环执行处理语句

EXIT 语句

在循环控制语句中，与循环条件无关，从循环控制中退出

<循环处理语句>

　　·

　　·

EXIT;

　　·

　　·

<循环处理语句>

执行 EXIT 后

不论条件如何，结束包含 EXIT 的循环控制

CONTINUE语句

在循环控制语句中，跳过循环进入下一个循环的执行中

<循环处理语句>

　　·

　　·

CONTINUE;

　　·

　　·

<循环处理语句>

执行CONTINUE时，无论条件如何，都会返回到含有CONTINUE的循环控制语句的开头，开始下一个循环。

* :表示插入半角空格。

▌程序执行结束语句

结束已编写的程序的执行。

控制语句

控制内容

RETURN语句

结束已编写的程序执行的控制语句

处理语句;

RETURN;

处理语句;

结束已编写的程序（模块、功能块、功能）的执行。

IF ～ 语句(条件分支 1)

IF 语句是根据 1 个条件式判断是否进行处理的、最简单的条件分支语句。

▌IF ～ THEN ～ END_IF

控制语句

格式说明

IF<条件式>THEN

　　<处理语句1>;

END_IF;

如果条件式成立，

则执行处理语句 1。

IF 语句结束

* :表示插入半角空格。

格式举例

说明

IFbFlag = TRUETHEN

　　uiValue := uiValue + 1;

END_IF;

如果 bFlag 为 ON

uiValue 加上常数 1。

 

•如果条件式的结果为比特型（返回布尔值）运算式，则可以在任何运算式中使用。

•处理语句是在 THEN～END_IF 之间编程的一系列的处理，可根据需要在多行间编程。

•IF 语句可以嵌套，因此通过在处理语句中再编程 IF 语句，可通过 AND（与）条件过滤的同时执行多重分支处理。

▌处理流程

 

(1)通过条件式判断是“TRUE”还是“FALSE”。

(2)如果条件式为“TRUE”，则执行 THEN 以下的处理语句 1，并跳到 END_IF 。

(3)如果条件式为“FALSE”，则跳到END_IF，不执行任何操作，并跳转到  END_IF 以后的处理。

IF ～ ELSE 语句(条件分支 2)

该语句是指通过 1 个条件式可分成 2 种处理方法的简单的条件分支语句。

▌IF ～ THEN ～ ELSE ～ END_IF

控制语句

格式说明

IF＜条件式＞THEN

　　＜处理语句1＞;

ELSE

　　＜处理语句2＞;

END_IF;

如果条件式成立，

则执行处理语句 1。

如果条件式不成立，

则执行处理语句 2。

IF 语句结束

* :表示插入半角空格。

格式举例

说明

IFbFlagTHEN

　　uiValue := 50;

ELSE

　　uiValue := 10;

END_IF;

如果 bFlag 为 ON，

则常数 50 存储到 uValue。

如果 bFlag 为 OFF，

则常数 10 存储到 uValue。

▌处理流程

 

(1)通过条件式判断是“TRUE”还是“FALSE”。

(2)如果条件式为“TRUE”，则执行 THEN 以下的处理语句 1，并跳到 END_IF （中间的运算式不执行）。

(3)如果条件式为“FALSE”，则不执行 THEN~ELSE 间的程序，而执行 ELSE 以下的处理语句2。

IF ～ ELSIF ～ 语句(条件分支 3)

如果不满足最初的条件式，则按照下一条件式重新进行判定。该条件分支语句可以具有多个条件。

▌IF ～ THEN ～ ELSIF ～ END_IF

控制语句

格式说明

IF＜条件式1＞THEN

 ＜处理语句1＞;

ELSIF＜条件式2＞THEN

 ＜处理语句2＞;

ELSIF＜条件式3＞THEN

 …

ELSE

 ＜处理语句n＞;

END_IF;

如果条件式 1 成立，

则执行处理语句 1。

如果条件式 1 成立，条件式 2 不成立，

则执行处理语句 2。

如果条件式 1，2 不成立，条件式 3 成立，*1

…

如果所有的条件式都不成立

则执行处理语句 n。

IF 语句至此结束。

* :表示插入半角空格。

*1:可以设定任意数量的条件式和处理语句。没有上限。

格式举例

说明

IFbFlagTHEN

　　uiValue := 1000;

ELSIFbFlag1THEN

　　uiValue := 1001;

ELSIFbFlag2THEN

　　uiValue := 1002;

ELSIFbFlag3THEN

　　uiValue := 1003;

ELSIFbFlag4THEN

　　uiValue := 1004;

ELSE

　　uiValue := 1005;

END_IF;

如果 bFlag 为 ON（bFlag =ON 的省略记法）

则 1000 存储在 uValue 中。

如果 bFlag 不为 ON，bFlag 1 为 ON，

则 1001 存储在 uValue中。

如果 bFlag、bFlag1 不为 ON，bFlag 2 为 ON，

则 1002 存储在 uValue中。

如果 bFlag、bFlag1、bFlag2 不为 ON，bFlag 3 为 ON，

则 1003 存储在 uValue中。

如果 bFlag、bFlag1、bFlag2、bFlag3 不为 ON，bFlag 4 为 ON，

则 1004 存储在 uValue中。

如果 bFlag 均不为 ON，

则 1005 存储在 uValue中。

 

•如果条件式的结果为比特型（返回布尔值）运算式，则可以在任何运算式中使用。

•处理语句是在控制语句之间编程的一系列的处理，可根据需要在多行间编程。

•ELSE IF语句与 IF 语句不同，可编程多行，但不是嵌套结构。

▌处理流程

 

(1)通过条件式 1 判断是“TRUE”还是“FALSE”。

(2)如果条件式 1 为“TRUE”，则执行 THEN 以下的处理语句 1，并跳到 END_IF（中间的运算式不执行）。

(3)如果条件式为“FALSE”，则以条件式 2 判断“TRUE”或“FALSE”。

(4)如果条件式 2 为“TRUE”，则执行处理语句 2，并跳到 END_IF（中间的运算式不执行）。

(5)如果条件式 2 为“FALSE”，则以条件式 3 判断“TRUE”或“FALSE”。

(6)如果条件式 3 为“TRUE”，则执行处理语句 3，并跳到 END_IF（中间的运算式不执行）。

(7)如果条件式 3 为“FALSE”，则以条件式 4 判断“TRUE”或“FALSE”。依次类推。

CASE ～ 语句(多分支控制)

▌CASE

该条件分支语句可根据存储在参考字数据中的值，选择处理。

控制语句

格式说明

CASE<字数据>OF

　　<定数1> : 处理语句1;

　　<定数2> : 处理语句2;

　　<定数3>,<定数4> : 处理语句3;

　　<定数5>..<定数6> : 处理语句4;

ELSE

　　<处理语句n>;

END_CASE;

基于字数据值的条件分支

如果常数 1 和字数据的值相等，则执行处理语句 1。

如果常数 2 和字数据的值相等，则执行处理语句 2。

字数据与常数3或常数4相等时，执行处理语句3。

常数5≤字数据≤常数6时，执行处理语句4。

常数和字数据的值是上述以外情况时，

执行处理语句n。

CASE 语句结束。

* :表示插入半角空格。

格式举例

说明

CASEuiCountOF

　　1 : uiValue := 1000;

　　2..10 : uiValue := 2000;

　　11,15,60 : uiValue := 3000;

　　101..65535 : uiValue := 4000;

ELSE

　　uiValue := 5000;

END_CASE;

以 uiCount 存储的值为基准。

如果 uiCount = 1，则 1000 存储在 uiValue 中。

如果 uiCount = 2～10，则 2000 存储在 uiValue 中。

如果 uiCount = 11 或 15 或 60，则 3000 存储在 uiValue 中。

如果uiCount的值为101以上时，则4000存储在 uiValue 中。

 

如果 uiCount 的值为上述以外的值，则 5000 存储在 uiValue 中。

▌处理流程

 

(1)声明作为以后条件值的判断基准的值。

(2)比较比较源和条件值 1，如果“比较源 = 条件值 1”成立，则执行处理语句 1。执行处理语句 1 后，跳到 END_CASE。

(3)如果“比较源 = 条件值 1”不成立，则跳至下一条件值比较。

(4)比较比较源和条件值 2，如果“比较源 = 条件值 2”成立，则执行处理语句 2。执行处理语句 2 后，跳到 END_CASE。

(5)如果“比较源 = 条件值 2”不成立，则转到下一个条件值比较。

(6)比较比较源和条件值 3，如果“比较源 = 条件值 3”成立，则执行处理语句 3。执行处理语句 3 后，跳到 END_CASE。

(7)如果““比较源 = 条件值 3”不成立，则执行处理语句 4。

 

•选择的常数可以是单一常数，或包括多项指定和范围指定的常数。

指定例）

1 : 单一指定（ 仅限 1）

3, 9, 12, 50 : 多个指定（ 3 或9 或12 或 50 ）

8..20 :范围指定（8 以上、20 以下的范围内）

•ELSE<处理语句>的编程可以省略。

但“如果 CASE 条件全不成立”的 CASE ELSE 可帮助解决发生意外事件的问题。

FOR ～ END_FOR 语句（指定次数的循环控制）

▌FOR～END_FOR

根据循环条件，决定处理语句执行次数的控制语句。

控制语句

格式说明

FOR<变量> := <初始值>TO<最终值>[BY<增量值>] DO

<处理语句>;

END_FOR;

1．在＜变量＞中存储＜默认值＞。

2．满足以下条件时，执行＜处理语句＞。不满足时，FOR语句结束。

·＜增量值＞ > 0时，＜变量＞≤＜最终值＞

·＜增量值＞ < 0时，＜变量＞≥＜最终值＞

3．＜变量＞加上＜增量值＞后返回2。

*1:[　]内的编程可以省略。如果省略，则作为 BY1 执行。
表示插入半角空格。

格式举例

说明

FORuiIdx := 0TO99BY5DO

　　uiValue := uiValue + 5;

END_FOR;

uiIdx 的值从 0 到超过 99，循环。

将 uiValue 加上 5 的结果存储到 uiValue。

在 uiIdx 的当前值加上增量值 5，返回 FOR~。

▌处理流程

 

(1)为变量赋值初始值。

(2)设置表示循环处理结束的最终值。另外，设置加在变量值上的增量值。

(3)判定变量值是否小于最终值。

(4)如果 (3) 的条件为“TRUE”，则执行处理语句。

(5)“FALSE”，则结束循环处理语句。

(6)将变量值加上增量值，返回 (3)。

WHILE 语句（前判定循环控制）

▌WHILE～END_WHILE

循环条件为“TRUE”期间，执行循环处理语句的控制语句。

控制语句

格式说明

条件式成立时、*1

执行处理语句。

返回句首（WHILE）。;

* :表示插入半角空格。

格式举例

▌处理流程

(2)判定 UNTIL 条件式是“TRUE”还是“FALSE”，并确认是否继续执行循环处理。

(3)如果条件式为“FALSE”，则返回 (1)。

(4)如果条件式为“TRUE”，则进入下一步。

使用控制语句时的注意事项

循环控制语句和处理时间

使用FOR语句、WHILE语句、REPEAT语句执行循环处理时，如果1次扫描中的循环次数变多，那么扫描时间也会变长。

扫描时间超过300ms时，CPU单元会发生扫描超时，转为PROGRAM模式后停止动作。

因此，需要在一定程度上控制1次扫描中的循环次数，以免发生扫描超时。

 

例如，执行20万次循环处理时，请不要在1次扫描中执行20万次循环处理，

而是按照　1000次循环处理/1次扫描×200次扫描　等方式，分成多次扫描执行循环处理。
